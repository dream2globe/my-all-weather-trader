# My All-Weather Trender (Dynamic Pension Portfolio)

본 프로젝트는 퇴직연금(IRP) 및 개인연금 계좌에서 운용하기 위한 **동적 자산 배분 및 가치평균법(Value Averaging) 자동화 백테스트 엔진**입니다.

## 🌟 핵심 전략 (Core Strategies)
1. **Value Averaging (가치평균법)**: SPY, KOSPI, GLD, SHV 등의 자산에 대해 매월 정해진 목표 복리 성장률(Target Growth Rate) 단가를 추종합니다.
   * 목표치보다 평가액이 부족하면 추가 매수, 초과하면 '초과분의 절반(50%)'만 매도하여 수익을 실현합니다.
   * 상승 랠리 시 곧바로 매도하지 않도록 KOSPI 등에 +5% 허용 밴드(Tolerance Band)를 적용합니다.
   * 동반 급락장에서 현금이 빠르게 고갈되는 것을 막기 위해 **동적 Max Cap(현재 3%로 강화)** 방어 로직이 적용되어 있습니다.
   * 매달 시드머니가 추가되는(Cash Injection) 월단위 적립식 투자 시나리오(동적 타겟 자동 우상향)를 완벽하게 지원합니다.
   * **동적 롤링 윈도우 적용**: 시스템 스스로 반기별로 과거 1~3년 치 데이터를 학습하여 VA 월 목표 성장률을 동적으로 갱신합니다 (코로나 시기 하락장 방어력 극대화).
2. **크립토 제외 확정 포트폴리오**: 초기에는 비트코인을 포함했으나 최종적으로 연금 직투의 본진 취지에 집중하여 비트코인(BTC)과 이더리움(ETH)을 취급 범주에서 완전 배제하고, 미국/국내 주식 시장에 집중투자(각 30%)합니다.
3. **최종 셋업 및 단기채(SHV) 닻(Anchor) 전략**: S&P500 30%, KOSPI 30%, 금 15%, 초단기채(SHV) 15%, 대기 현금 10% 구성으로 완료. 장기채의 금리 리스크 차단을 위해 방어력이 가장 뛰어난 1년 미만 초단기채(SHV)를 채택하여 포트폴리오의 안정성을 높입니다.

## 🚀 IRP 직투용 "투트랙(Two-Track)" 가이드
분석 엔진은 깊고 긴 과거 데이터(20~30년)를 가진 **미국 본토 시세**를 기반으로 백테스트 및 목표가를 정밀하게 주간 단위로 계산하지만, **실제 IRP 계좌 매매 시에는 100% 호환되는 국내 상장 ETF를 대리 매수**합니다. 

| 계산 기준 (미국 원조 ETF) | ➡️ | 실전 매수 종목 (국내 IRP용 ETF) | 비수익/특징 |
|:---|:---:|:---|:---|
| **SPY** (S&P 500) | ➡️ | **KODEX S&P500 TR** (379800) | 배당 자동재투자, 환노출 |
| **KOSPI** | ➡️ | **KODEX 200** (069500) | 유동성 풍부 (위성 전략) |
| **GLD** (금 현물) | ➡️ | **ACE KRX금현물** (411060) | 퇴직연금 내 유일한 금현물 상품 |
| **SHV** (1년 미만 단기채) | ➡️ | **TIGER 미국초단기(3개월)국채** (482730) | 현금성 대기 자금 파킹 |

> (매번 월요일 오전에 자동화 봇이 `SPY 500만 원 매수` 시그널을 주면, IRP 증권사 앱을 켜서 KODEX S&P500 TR을 500만 원어치 매수 버튼을 누르시면 됩니다.)

## ⚙️ 설치 및 실행 방법

### 1단계: 환경 설정 (`uv` 패키지 매니저 권장)
빠르고 가벼운 파이썬 패키지 매니저인 `uv`를 이용하여 숨김 폴더(`.venv`)에 가상환경을 생성하고 라이브러리를 설치합니다.
```bash
# 가상환경 생성 및 활성화
uv venv .venv
source .venv/bin/activate

# 의존성 패키지 초고속 설치
uv pip install -r requirements.txt
```

### 2단계: 백테스트 시뮬레이션 엔진 구동
`main.py`를 실행하여 포트폴리오 유니버스의 전체 과거 데이터를 백테스트하고, VA 타겟의 최적화 수준을 점검합니다. 
```bash
python main.py
```

### 3단계: [중요] 실전 계좌 연동 주간 매매 계획 산출
실제 연금 계좌의 체결 내역을 바탕으로 매주 월요일 '실행 계획'을 도출합니다.
1. `data/actual_trades.csv` 표에 주간 매매 내역을 수기로 1줄 입력합니다.
2. 아래 명령어로 이번 주 매수/매도할 티커와 수량 권고안을 마크다운 리포트로 뽑아냅니다.
```bash
python generate_live_plan.py
```
엔진 실행이 정상 종료되면, `reports/` 폴더 하위에 다음의 결과물들이 자동 추출됩니다.
* **Tear Sheet (`.txt`)**: 연평균수익률(CAGR), Max MDD, Sharpe 비율 등 종합 지표 요약
* **지표 그래프 (`.png`)**: 자산 성장 리포트(Growth), 하락폭(MDD), 시간에 따른 자산별 비율(Allocation) 
* **Signal Scatter Plot**: 각 주가 차트 표면 위에 매수(Buy) / 매도(Sell)가 정교하게 타점(Scatter)된 시각화 리포트
* **거래 장부 (`.csv`)**: 언제, 무슨 종목을, 얼마의 가격과 수량, 어느정도 수수료를 떼고 매매했는지에 대한 엔진 시뮬레이션 체결 상세 로그

### 4단계: (선택) 단위 테스트(Unit Test)를 통한 검증
각 모듈(엔진 체결기 및 시그널 생성기)의 안전성을 독립적으로 검증합니다.
```bash
pytest tests/ -v
```

## 📅 매주 주말 정기 점검 가이드 (Weekly Check-up Routine)
본 시스템은 가치평균법을 기반으로 작동하므로 매일 계좌를 들여다볼 필요가 없습니다. 다음 3단계 루틴을 통해 월요일 매매를 준비하세요.

### 1️⃣ 이번 주 체결 내역 업데이트 (금/토요일)
평일 동안 IRP 계좌에서 발생한 실제 매수/매도/배당금 내역을 `actual_trades.csv`에 한 줄씩 기록합니다.
* **경로**: `data/actual_trades.csv`
* **예시**: `2026-03-13,SP500,BUY,662.95,2400.0,0.0`
* 만약 계좌에 추가 현금을 투입했다면, 해당 금액 역시 거래 내역(`Action=DEPOSIT` 등 시스템 정의에 맞춰)으로 남겨 계좌 잔고가 일치하도록 맞춥니다.

### 2️⃣ 실전 매매 계획 리포트 생성 (일요일)
주말 동안 여유가 있을 때 가상환경을 켜고 리포트 생성 명령어를 실행합니다.
```bash
source .venv/bin/activate
python generate_live_plan.py
```
* 이 작업은 엔진이 스스로 최신 주가를 웹에서 긁어와 현재 내 계좌의 평가액을 계산합니다.
* 그런 다음 이번 달의 'VA 목표 타겟 평가액'과 비교하여 차주 월요일에 얼마를 추가 매수(혹은 매도)해야 하는지 정확한 `KRW` 금액 액수를 뽑아냅니다.

### 3️⃣ 리포트 확인 및 예약 매수 (일/월요일)
* `reports/live_trade_plan_YYYYMMDD.md` 파일이 새로 생성된 것을 확인합니다.
* 리포트 안에 적힌 **자산별 매매 권고 금액**을 확인합니다.
* 월요일 출근 전이나 주말 저녁에 증권사 앱을 열고, 리포트에서 지시한 수량만큼 국내 상장 ETF(예: KODEX S&P500 TR)를 **시장가 혹은 여유 있는 지정가로 예약 매수** 걸어둡니다.

> **💡 Note**: 시장이 너무 크게 하락해서 목표치와의 갭이 너무 벌어졌더라도 안심하세요. `settings.py`에 적용된 동적 Max Cap(현재 3%)이 "이번 주는 이만큼만 사고 다음 주에 또 사자"라고 브레이크를 걸어줍니다.

## 📂 프로젝트 핵심 구조
* `src/backtest/engine.py`: 시뮬레이션 엔진 및 체결 제약(소수점 매매 컷, 현금 부족 시 영끌, 수수료) 핸들러
* `src/config/settings.py`: 자산 분배 비중, 전략 파라미터(성장률, 타겟변동성) 등 중앙값 관리 파일 (Pydantic BaseSettings)
* `src/strategies/`: 매매 시그널을 생산하는 코어 논리 (`value_averaging.py`, `base_strategy.py`)
* `src/visualization/plot_utils.py`: Scatter plot을 포함한 리포트 생성용 Seaborn/Matplotlib 시각화 유틸
* `reports/`: 백테스트 결과물이 생성/적재되는 목적 경로 (gitignore 권장이나 마일스톤 리포트는 포함)
* `tests/`: Pytest에 기반한 각 모듈의 단위 테스트 케이스
